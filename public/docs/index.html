Get started

npm install
npm run dev     # editor at http://localhost:5173 (+ local render server)

Build by dragging Presets, Elements, and Effects from the left panel (use the 🔎 search box to find things fast). Tracks are layers — the top track renders on top. Select a clip to edit every parameter in the Inspector; drag on the canvas to reposition it. Everything autosaves to your browser, and Save/Open exchanges portable JSON files.

🧩 The HTML element

The HTML element turns any HTML + CSS you can write into an animatable clip — chat windows, app mockups, pricing cards, buttons, browser frames. It's how you make HyperFrames-style product launch videos (see examples/hyperframe-style-launch.json). Because both the preview and the renderer are Chrome, what you paste is exactly what exports.

How it works

  • The clip is a box of Width × Height canvas pixels, centered on X/Y. Your markup fills that box (make your root element width:100%;height:100%).
  • Content scale zooms the markup inside the box without reflowing it — design at a comfortable size, then scale.
  • Edit the markup in the Inspector textarea; the ✨ Format HTML button pretty-prints it.
  • Animate the whole box with any effects: slide it in, wipe it, shake it, glitch it, move it along a path.

Rules & gotchas

  • Inline styles only (style="…"). A <style> tag technically works but applies globally — if two HTML clips define the same class, they'll fight. Inline styles can't collide.
  • No JavaScript. Scripts don't execute (by design — animation belongs to effects, so it's deterministic per frame). CSS animations/transitions also won't render correctly on export because frames are captured out of real time: use the editor's effects for motion.
  • Images inside HTML: allowed via <img> with absolute URLs or data-URIs. Prefer the dedicated Image element when the image needs its own animation.
  • Fonts: system fonts are safe (Helvetica, Arial, Georgia, 'Courier New'…). Webfonts loaded by URL aren't wired up yet.
  • The box clips its content (overflow hidden) — nothing bleeds outside the Width×Height.

Recipe: fake a UI interaction

  1. Add an HTML clip with your mockup (button, form, timeline…).
  2. Add a 🖱️ Cursor clip on a track above it. Give it a Motion Path (draw it on the canvas) ending on the control.
  3. Add 👆 Click Pulse to the cursor at the arrival frame, and a subtle Click Pulse (depth ≈ 0.06) to the HTML clip a frame later — the "press" reads instantly.
  4. React to the click with other clips: a text pop, a color flash, a scene cut.

Effects

Drag any effect onto a clip; stack freely. Every parameter is editable in the Inspector.

  • Entrances: Fade In, Slide In, Zoom In, Spring Pop, Spin In, Wipe In, Blur In, Bounce Drop, Typewriter, Letter Pop
  • Exits: Fade Out, Slide Out, Zoom Out, Wipe Out
  • Loops: Rotate, Pulse, Wiggle, Camera Shake, Light Sweep, Strobe
  • Style: Ken Burns, Drift, Glow, Drop Shadow, RGB Split, Glitch, Transform, Motion Path

Animation curves

Movement effects have a Curve parameter with a live graph: easeIn/Out/InOut, linear, bounceOut (ball landing), elastic (springy settle), overshoot (goes past, comes back).

🏀 Bouncing ball: just add the Bounce Drop effect — or use Transform (animate Y) with curve bounceOut.

Transform (keyframe-style)

Animates position / scale / rotation / opacity from A to B over any frame range inside the clip. Stack several Transforms with different Start at frames for multi-step choreography.

Motion paths

Add the Motion Path effect to a clip, then either:

  1. Draw it: click ✏️ Draw path on canvas in the Inspector, click points on the preview (the green dot is the start), then Enter or double-click to finish. The element moves to your first point and follows the polyline.
  2. Type it: paste any SVG path — e.g. M 0 0 C 150 -250 450 -250 600 0 (arc) or M 0 0 Q 200 -150 400 0 T 800 0 (s-curve). Coordinates are relative to the element.

Enable Rotate to follow path for planes, arrows, and fish.

Presets

Double-click (or drag onto the timeline) to insert ready-made, canvas-sized animations: Title Intro, Lower Third, Glitch Title, Ken Burns Photo, Social Pop, Stat Counter, News Ticker, Quote Card, End Card. After inserting they're normal clips — retext, recolor, retime.

Saving & export

  • Autosave to the browser; 💾 Save / 📂 Open for JSON files; ✚ New for a blank project.
  • ⬇ Export MP4 renders on the local render server with progress — output lands in out/ and downloads from the dialog.
  • Any size up to 4K60: pick a Size preset (4K UHD, DCI, vertical 4K, square, 4:5, ultrawide) or type custom dimensions, choose FPS (24–60), then export at 0.5× / 1× / 2× scale.
  • CLI: npm run render -- project.json out/video.mp4
